# Functional Requirements

All of these requirements are specified and related to NFR into requirement_specification.md file. 

## FR1: Landing Page
- FR1.1: The web application must display a list of regions, along with a dashboard of information for each region and below it, the vineyards belonging to the geographic filter.
- FR1.2: The web application must display a dashboard of information for each region.
- FR1.3: The web application must allow navigation between vineyard and wine sections through a navigation bar.
- FR1.4: The web application must allow filtering vineyards according to categories (to be defined).
- FR1.5: The web application must allow filtering wines according to categories (to be defined).
- FR1.6: The web application must allow filtering regions according to categories (to be defined).

## FR2: Vineyard Page
- FR2.1: The vineyard page must display the wines belonging to the vineyard.
- FR2.2: The vineyard page must display the reviews made to the vineyard.
- FR2.3: The vineyard page must allow users to create a review of the vineyard.
- FR2.4: The web application must display the information related to the vineyard.

## FR3: Wine Catalog
- FR3.1: The web application must allow filtering wines belonging to the vineyard according to categories (to be defined).
- FR3.2: The application must display relevant metrics for each wine.
- FR3.3: When selecting a wine, the user must be able to view its detailed information.

## FR4: Wine Selected
- FR4.1: The web application must display the general information of the selected wine.
- FR4.2: The web application must display the technical information of the selected wine.

## FR5: Wine Reviews
- FR5.1: The application must allow the user to create a review by selecting between amateur or expert format.
- FR5.2: The amateur user must be able to enter their review without needing to fill in technical fields.
- FR5.3: The expert or certified critic user must be able to enter a critique that satisfies all technical aspects.
