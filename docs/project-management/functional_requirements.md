# Functional Requirements

**Geographic / Navigation**
- **FR1.1:** The web application must display, as the main entry view, a list of countries with their aggregated wine metrics, allowing the user to get a general overview of each country's performance before diving into regions and vineyards.
- **FR1.2:** The web application must have a persistent navigation bar that allows the user to directly access the Countries, Vineyards and Wines sections from anywhere in the application, without needing to follow a hierarchical flow.
- **FR1.3:** The web application must allow the user to filter the vineyard listing by location, production and review performance categories, so they can find vineyards of interest without browsing the full list.
- **FR1.4:** The web application must allow the user to sort the vineyard listing by multiple criteria simultaneously, with user-defined priority and configurable direction for each criterion.
- **FR1.5:** The web application must allow the user to filter the wine listing by origin, wine characteristics, price and review performance categories, so they can find wines of interest without browsing the full list.
- **FR1.6:** The web application must allow the user to sort the wine listing by multiple criteria simultaneously, with user-defined priority and configurable direction for each criterion.

**Vineyard Page**
- **FR2.1:** The vineyard page must display general information, geographic characteristics and aggregated metrics of the vineyard, including a summary of its wines and the reviews received by both the vineyard and its wines.
- **FR2.2:** The vineyard page must display the list of wines commercialized by the vineyard, allowing the user to explore the vineyard's products without leaving its dashboard.
- **FR2.3:** The vineyard page must display the reviews made to the vineyard as an entity, allowing the user to browse and filter them by reviewer type, similarly to the main reviews section but restricted to the current vineyard.
- **FR2.4:** The vineyard page must allow the user to create a review of the vineyard through a form that varies according to the selected reviewer type: enthusiast, expert or organization.

**Wine Page**
- **FR3.1:** The web application must display a dashboard with the complete information and metrics of each wine, satisfying the needs of both an enthusiast user and an expert critic.
- **FR3.2:** The wine dashboard must display the reviews made to that specific wine, allowing the user to browse and filter them by reviewer type.
- **FR3.3:** The wine dashboard must allow the user to create a review of the wine through a form that varies according to the selected reviewer type: enthusiast, expert or organization.
