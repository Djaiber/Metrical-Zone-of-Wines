## Description

**Statement:**
> The wine dashboard must allow the user to create a review of the wine through a form that varies according to the selected reviewer type: amateur, expert, or organization.

**Involved Actor:**
Any visitor to the wine dashboard. No authentication is required. The reviewer type is selected by the user from the form, without formal identity verification.

---

## Conditions

**Precondition:**
- The user is on a wine's dashboard.
- The review creation form is visible or accessible from the wine's reviews section.

**Postcondition:**
The review is stored in MongoDB as a document with `wine_id` corresponding to the current wine and `reviewer_type` according to the user's selection. The wine's metrics in `wine_metrics` and the vineyard's in `vineyard_metrics` are updated by the backend upon receiving the new review.

## Acceptance Criteria

### Reviewer type selection
- [ ] **AC1:** The form presents the user with three reviewer type options: **Amateur**, **Expert** or **Organization** before showing the specific fields.
- [ ] **AC2:** When selecting the reviewer type, the form shows only the fields corresponding to that profile.
- [ ] **AC3:** The reviewer type cannot be modified once the form filling has started without restarting it.

### Form — Amateur
- [ ] **AC4:** The amateur form includes the following mandatory fields: reviewer name, overall score (scale 0–100), free description of the experience with the wine, consumption occasion, and recommendation indicator (yes/no).
- [ ] **AC5:** The `consumption_occasion` field is presented as a list of predefined options: celebration, food pairing, casual consumption, tasting, gift.

### Form — Expert and Organization
- [ ] **AC6:** The expert or organization form includes the following mandatory fields: reviewer name, occupation, organization, years of experience, overall score (scale 0–100), review year, dimension scores — color, aroma, taste, finish, and structure (each on scale 0–10), tasting notes, and suggested pairing.
- [ ] **AC7:** The dimension scores (color, aroma, taste, finish, structure) are independent of each other and do not condition the value of `score_overall`.

### Validations
- [ ] **AC8:** All form fields are mandatory — the system prevents submission if any is empty, showing an error message per missing field.
- [ ] **AC9:** The `score_overall` field accepts only decimal values between 0 and 100. Values outside the range are rejected with an error message.
- [ ] **AC10:** The dimension scores accept only decimal values between 0 and 10. Values outside the range are rejected with an error message.
- [ ] **AC11:** The `years_experience` field accepts only positive integer values greater than 0.
- [ ] **AC12:** The `review_year` field cannot be a future year.

### Submission and confirmation
- [ ] **AC13:** When the form is submitted correctly, the system shows a confirmation message and the review appears in the wine's listing without needing to reload the page.
- [ ] **AC14:** If an error occurs when saving the review, the system shows a descriptive error message and preserves the entered data so the user can retry.
- [ ] **AC15:** The document stored in MongoDB includes the `submitted_at` field automatically generated by the backend — this field is not visible or editable by the user.

## Generated MongoDB Document Structure

**Amateur:**
```json
{
  "reviewer_type": "enthusiast",
  "reviewer_name": "string",
  "wine_id": "int",
  "score_overall": "decimal (0–100)",
  "experience_description": "string",
  "consumption_occasion": "enum (celebration | food pairing | casual consumption | tasting | gift)",
  "would_recommend": "boolean",
  "submitted_at": "ISODate (generated by backend)"
}
```

**Expert / Organization:**
```json
{
  "reviewer_type": "expert",
  "reviewer_name": "string",
  "occupation": "string",
  "organization": "string",
  "years_experience": "int",
  "wine_id": "int",
  "score_overall": "decimal (0–100)",
  "review_year": "int",
  "scores": {
    "color": "decimal (0–10)",
    "aroma": "decimal (0–10)",
    "taste": "decimal (0–10)",
    "finish": "decimal (0–10)",
    "structure": "decimal (0–10)"
  },
  "tasting_notes": "string",
  "pairing_suggestions": "string",
  "submitted_at": "ISODate (generated by backend)"
}
```

## Notes and Change History

- Wine reviews never include `vineyard_id` — they are exclusive to the wine as an entity.
- The dimension scores (`scores`) are exclusive to expert wine reviews — they do not exist in vineyard reviews.
- `submitted_at` is generated by the backend at the time of saving, not by the frontend.
- The backend is responsible for updating `wine_metrics` and `vineyard_metrics` in MySQL after receiving and storing a new review in MongoDB.
- The dimension scores do not automatically average towards `score_overall` — they are independent fields that the expert fills separately.
