REGLAS DE ORO ANTES DE PROBAR:
1. Reemplacen los IDs anidados en las relaciones. Si el pais les queda con el id 12,
   usen ese id en la region, pero revisen que id les da la region para usarlo
   en el viñedo, y asi sucesivamente.
2. Lean los comentarios (//) en los JSONs para saber que opciones son validas
   en cada Enum y evitar errores 500 por "Data truncated".

-------------------------------------------------------------------------------
1. PAISES (Country)
-------------------------------------------------------------------------------
METODO: POST
URL: /api/countries
BODY:
{
  "name": "New Zealand",
  "code": "NZ"
  // OJO: 'code' debe ser exacto de 2 letras y UNICO.
}

-------------------------------------------------------------------------------
2. VARIEDAD DE UVA (GrapeVariety)
-------------------------------------------------------------------------------
METODO: POST
URL: /api/grape-varieties
BODY:
{
  "name": "Sauvignon Blanc Exclusive Version",
  "color": "White",
  // Opciones validas para color: 'Red', 'White', 'Rosé'
  "originCountry": "France"
}

-------------------------------------------------------------------------------
3. REGIONES (Region)
-------------------------------------------------------------------------------
METODO: POST
URL: /api/regions
BODY:
{
  "name": "Marlborough",
  "description": "Region famosa por sus vinos blancos de clima frio.",
  "climateType": "Oceanic",
  // Opciones: 'Continental', 'Mediterranean', 'Oceanic', 'Semi-arid', 'Tropical', 'Alpine'
  "wineStyleProfile": "Crisp whites",
  // Opciones: 'Bold reds', 'Crisp whites', 'Elegant reds', 'Sparkling', 'Sweet & dessert', 'Rosé'
  "country": { "id": 13 } // Pongan el ID real de New Zealand
}

-------------------------------------------------------------------------------
4. METRICAS DE REGION (RegionMetric)
-------------------------------------------------------------------------------
METODO: POST
URL: /api/region-metrics

{

  "totalReviews": 12000,
  "avgScore": 90.5,
