# 🍷 Metrical Zone of Wines

**Metrical Zone of Wines** is a full-stack data analytics platform designed to explore and evaluate wine regions, vineyards, and wine reviews using a hybrid **SQL + NoSQL** architecture.

The system combines **MySQL for structured relational data** with **MongoDB for flexible document storage**, enabling advanced analytics on wine quality, regional performance, and user feedback.

This project is developed as part of a **Database Systems II academic project**, demonstrating practical database engineering concepts such as relational modeling, stored procedures, triggers, transaction management, and NoSQL document design.

---

# 🚀 Key Features

- Wine region analytics dashboard
- Hybrid **Relational + NoSQL database architecture**
- Stored procedures and database-level business logic
- Transaction management and exception handling
- MongoDB document-based review storage
- Full-stack web application architecture
- Infrastructure deployed on Linux virtual machines
- Database version control using **Flyway**
- Infrastructure automation with **Terraform**
- Configuration management using **Ansible**
- CI/CD automation with **GitHub Actions**

# 🗄 Database Design

The relational database includes:

- **8+ normalized entities**
- **Primary and foreign key constraints**
- **Stored procedures**
- **SQL functions**
- **Triggers**
- **Transaction control (COMMIT / ROLLBACK)**
- **Exception handling**

MongoDB is used to store wine review documents that include:

- tasting notes
- optional attributes
- user review metadata
- flexible document schema
